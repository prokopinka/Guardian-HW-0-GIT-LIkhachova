HW 0: GIT

git config --global user.name blabla - установить имя пользователя, которое будет использоваться гитом

git config --global user.email blabal@bla - установить имейл пользователя, которое будет использоваться гитом

git init - инициализировать локальный репозиторий

git status - показать статус рабочего дерева

git add - выбрать изменения, которые будут включены в следующий коммит

git commit - создать слепок изменений файлов в гит репозитории

git log - просмотреть историю коммитов

git checkout - переключиться на другую ветку/ комит

git branch - создать дополнительную ветку изменений в репозитории

git push - отправить локальные изменения в удаленный репозиторий

git pull - стянуть на локальный репозитория изменения с удаленного

git merge - обьединить последние изменения в репозиториях, прежде чем отправить коммит в удаленный репозиторий

git clone - скопировать удаленный репозиторий в локальный


.gitignore файл - имеет информацию о файлах, изменения которы намеренно не нужно трекать. То есть, гит их игнорирует

Advanced:

git revert - отменит коммит, который уже был отправлен в удаленный репозиторий

git reset - удалить предыдущий коммит из истории

git rebase - переметить часть с коммитов с одной ветки на другую

git cherry-pick - переместить коммит, отправленный в неправильную ветку

git stash - временно хранить изменения, которые нежелательно отправлять в удаленный репозиторий
